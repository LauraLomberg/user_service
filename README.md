# О проекте
User Service — часть микросервисного приложения CorporationX, социальной сети для совместной работы над проектами и стартапами. Проект состоит из независимых сервисов, связанных через асинхронный обмен событиями (Kafka, Redis). Основная цель — дать пользователям возможность создавать команды, публиковать контент, взаимодействовать друг с другом и развивать свои проекты.  
Система построена на event-driven архитектуре, что обеспечивает масштабируемость, устойчивость и асинхронную обработку нагрузок.

---

# Что делает User Service
* Регистрация и хранение профилей пользователей  
* Управление подписками  
* Запросы на менторство  
* Постановка целей  
* Создание навыков пользователей  
* Рекомендации и обратная связь между пользователями

# Что я реализовала

### Сервис рекомендаций
Сервис рекомендаций отвечает за возможность оставлять и получать обратную связь между пользователями, с которыми они вместе работали.

**Реализовано:**
* Создание, обновление и удаление рекомендаций с соблюдением бизнес-правил:  
  * один пользователь может оставить рекомендацию другому не чаще одного раза в 6 месяцев  
  * пользователь не может написать рекомендацию самому себе  
  * редактировать и удалять рекомендацию может только её автор  
* Получение рекомендаций по фильтрам (по автору, получателю, содержимому)  
* Валидация данных с использованием Jakarta-аннотаций  
* Логирование действий через Slf4j для мониторинга  
* Использование DTO и MapStruct для преобразования данных между слоями приложения  
* Обработка исключений  
* Unit-тесты, покрытие >70%  

---

### Сервис подписки
Сервис подписки отвечает за управление подписками пользователей и генерацию событий при подписке, чтобы другие сервисы могли реагировать на изменения.

**Реализовано:**
* Создание события о подписке на пользователя  
* Настройка асинхронной отправки событий через Redis в сервис нотификаций, чтобы подписчик получал уведомление о новых событиях  
* Обеспечение корректной работы бизнес-логики и интеграции с другими сервисами
